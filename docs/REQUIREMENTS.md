## 1. 项目概述

### 1.1 项目背景
用户在日常生活中需要管理大量任务，同时希望能够提高专注效率。传统的任务管理工具仅提供基础的增删改查功能，缺乏对用户工作方式的深入分析和智能引导。本项目旨在开发一个智能专注追踪应用，结合任务管理、专注计时和事后分析功能，帮助用户建立更好的工作习惯。

### 1.2 核心价值
- **完整的任务管理**：支持清单、任务、子任务的三级结构，便于组织复杂工作
- **习惯养成**：科学的打卡机制，帮助用户培养长期好习惯
- **专注计时**：番茄工作法，帮助用户保持专注
- **事后分析**：通过数据和图表了解自己的工作模式
- **智能提示**：在适当的时候给出有用的建议（而非主动干预）

## 3. 开发优先级

> **重要说明**：AI 智能模块作为最后一阶段开发，确保基础功能稳定、数据积累充分后再集成 AI 能力。

### 3.1 开发顺序

| 阶段 | 内容 | 时间 | 优先级 |
|-----|------|------|-------|
| **P0 核心功能** | 项目初始化、数据库设计、任务 CRUD、番茄钟计时器、本地数据持久化、系统通知 | 第 1-2 周 | 最高 |
| **P1 习惯模块** | 习惯 CRUD、打卡、统计、热力图、跳过机制 | 第 2-3 周 | 高 |
| **P2 数据可视化** | 统计卡片、热力图、时间轴、趋势图表、详情页 | 第 3-4 周 | 高 |
| **P3 云同步** | WebDAV 配置、压缩上传、下载解压、自动同步、冲突解决 | 第 4-5 周 | 中 |
| **P4 AI 智能模块** | AI 触发引擎、场景化提示、分析报告、周期性分析 | 第 5-6 周 | 低（最后开发） |

### 3.2 为什么 AI 模块最后开发？

1. **数据基础**：需要积累足够的任务、番茄钟、习惯数据才能提供有意义的 AI 分析
2. **用户体验**：确保基础功能稳定可靠，AI 才是锦上添花而非干扰
3. **成本控制**：避免早期频繁调用 AI API 增加开发成本
4. **需求验证**：先验证用户对基础功能的使用情况，再决定 AI 功能的具体价值

## 4. 功能需求

### 3.1 任务管理

#### 3.1.1 任务创建

**用户操作流程**：
1. 用户在顶部输入框输入任务标题
2. 用户按 Enter 键或点击"添加"按钮
3. 任务被创建并立即显示在任务列表中

**用户反馈**：

- 创建成功后，任务列表自动刷新
- 输入框清空，光标保持在输入框（可连续创建）
- 创建失败时（如标题为空或只有空白字符），不做任何操作

**默认配置**：
- 新任务使用默认优先级："无"
- 新任务**默认不设置截止时间**（用户需要时手动添加）
- 新任务默认不设置提醒
- 新任务默认不重复
- 新任务默认番茄钟：1 个番茄，25 分钟

**AI 智能提示（代码补全式）**：（最后阶段开发）

1. 用户在输入框中输入任务标题
2. 系统在输入框的光标后边灰色显示 AI 任务名字建议补全（防抖机制：用户停止输入 500ms 后触发，最少输入 3 个字符）
3. 用户可按 **右箭头键（→）** 应用 AI 建议
4. 用户可按 **Esc 键**取消 AI 建议
5. 建议内容会自动填充到表单

**用户反馈**：

- AI 建议以灰色文字显示，不干扰用户输入
- 右箭头键快速应用建议（避免与 Tab 键表单导航冲突）
- Esc 键取消建议，不影响已输入内容

---

#### 3.1.2 任务编辑

**用户操作流程**：

1. 用户点击任务卡片
2. 右侧弹出任务详情编辑侧边栏，显示所有可编辑项
3. 用户可修改：
   - **任务名称**（必填）
   - **优先级**（无/低/中/高）
   - **本次截止日期**（日期选择器，按天）
     - 用于本次任务逾期判定
     - 开启重复时，该日期将作为重复锚点
   - **所属清单**（下拉选择已有的或者输入名字创建新的）
   - **开始提醒时间**（日期时间选择器）
   - **是否重复**（不重复/每天/每周/每月）
   - **备注**（多行文本输入）
   - **番茄钟设置**：
     - 番茄个数（数字输入）
     - 自定义每个番茄的时间（分钟输入）
4. 用户点击"保存"按钮
5. 任务信息更新，任务列表刷新

**番茄钟运行中的编辑限制**：
- 番茄钟运行中**允许编辑**任务的所有字段，包括番茄钟设置
- 编辑番茄钟设置时，当前正在运行的番茄钟不受影响，新设置将在下一个番茄钟生效

**用户反馈**：
- 保存成功后，任务信息立即更新
- 用户点击"取消"或点击侧边栏外区域可关闭编辑
- 有未保存更改时，点击外部区域弹出确认对话框

**重复任务规则**：
- 重复任务**必须**设置本次截止日期，否则不允许开启重复
- 系统基于重复规则（daily/weekdays/weekly/monthly/custom）和锚点日期自动计算应生成的实例日期
- 应用启动时自动补齐缺失实例（从 `last_generated_date + 1` 到当天）
- 若同一规则在同一 `due_at` 已存在任务，则不重复创建
- 任务被标记为完成/取消不会直接触发“立即生成下一条”，实例创建由规则调度统一处理
- 新生成任务继承规则模板属性（优先级、番茄设置、所属清单、提醒等）
- 用户取消重复设置时，仅停用规则；已创建任务不受影响

#### 3.1.3 任务删除

**用户操作流程**：
1. 用户在任务列表中点击"删除任务"按钮（或批量选择后点击"删除"）
2. 任务**直接删除**，无需确认对话框（依赖撤销机制）
3. 相关子任务也被级联删除
4. 系统自动记录删除操作到日志（仅记录任务标题和删除时间，无需用户填写原因）
5. 任务列表刷新
6. 底部显示 Toast 提示："任务已删除 [撤销]"（10秒内可撤销）

**撤销机制**：
- 删除后 10 秒内，用户可点击 Toast 中的"撤销"按钮恢复任务
- 撤销恢复任务及其所有子任务
- 超过 10 秒后，删除不可撤销

**用户反馈**：
- 删除成功后，任务从列表中移除
- 删除操作自动记录，不打断用户流程
- 无确认对话框，操作更流畅

#### 3.1.4 任务完成

**用户操作流程**：
1. 用户点击任务卡片上的"完成"按钮
2. 任务状态变为"已完成"
3. 任务在列表中标记为已完成（可分出一个列表列出完成的任务）
4. 完成时间被记录
5. 系统在后台计算预估时间和实际完成时间的偏差（静默记录，不打断用户）

**时间偏差分析**：
1. 系统计算：
   - 预估时间 = 番茄个数 × 每个番茄时间
   - 实际时间 = 所有**已完成**状态的番茄钟实际总时长（不包括放弃的番茄钟）
   - 偏差百分比 = |实际时间 - 预估时间| / 预估时间 × 100%
2. 偏差数据**静默记录**到任务完成日志，不弹出对话框打断用户
3. 如果预估番茄数为 0，跳过偏差分析

**任务总结（可选，用户主动填写）**：
- 用户可在**任务详情页**查看已完成任务，主动添加心得体会
- 不在完成时弹出对话框，避免打断用户工作流
- 任务详情页显示：预估 vs 实际时间对比、偏差百分比
- 用户可随时补充心得体会和偏差原因

**用户反馈**：
- 任务状态立即更新，**不弹出任何对话框**
- 底部 Toast 提示："任务已完成 ✓"
- 如果任务关联了预估番茄数，Toast 额外显示实际 vs 预估对比（如"25分钟/30分钟"）

#### 3.1.5 任务排序

**用户排序操作流程**：

1. 用户进入任意任务列表（今天 / 明天 / 最近7天 / 全部 / 清单）
2. 直接拖拽任务卡片调整顺序
3. 松开后立即生效并持久化

**规则**：
- 所有未完成任务列表都支持拖拽排序（已完成列表除外）
- 仅支持拖拽排序，不提供创建时间/截止时间/优先级等复杂排序选项
- 拖拽失败时保持原顺序并提示“排序失败，请重试”
- 数据库使用 `sort_order` 保存拖拽后的顺序

#### 3.1.6 清单管理

**用户新增操作流程**：

1. 用户进入任务页面
2. 左侧边栏显示所有清单列表
3. 用户点击"新建清单"
4. 输入清单名称、选择颜色、可选图标
5. 保存清单
6. 清单出现在清单列表中

**用户反馈**：
- 清单列表更新
- 任务创建时可选择新清单
- 清单颜色在任务列表中显示

**系统默认清单**：
- 系统内置默认清单"收集箱"，用于存放未分配清单的任务
- "收集箱"**不可删除、不可改名**

#### 3.1.7 子任务管理

**用户创建子任务流程**：
1. 用户在任务详情侧边栏中点击"添加子任务"
2. 输入子任务标题
3. 按 Enter 键或点击"添加"按钮
4. 子任务显示在父任务下方（缩进显示）

**子任务嵌套限制**：
- 子任务**仅支持一层嵌套**（任务 → 子任务）
- 子任务不能拥有自己的子任务
- 每个父任务最多支持 **50 个**子任务

**用户编辑子任务流程**：
1. 用户点击子任务
2. 可编辑：子任务标题、完成状态
3. 子任务不支持独立的番茄钟设置（使用父任务的设置）

**子任务与父任务关系**：
- 删除父任务时，所有子任务同时删除
- 父任务完成时，如有未完成的子任务，系统自动将这些子任务标记为"随父任务完成"并一并完成（不弹对话框）
- 所有子任务完成时，系统在父任务卡片/详情页显示**非阻塞提示**（如提示条/Toast）：点击即可完成父任务（不弹对话框）
- 子任务不能独立于父任务存在

**用户反馈**：
- 子任务以缩进列表形式显示在父任务下
- 子任务完成进度显示为"已完成/总数"（如 2/5）

### 3.2 番茄钟计时器

#### 3.2.0 计时器状态持久化与恢复

**状态持久化**：
- 系统每秒将计时器状态持久化到本地存储
- 持久化内容包括：当前任务ID、剩余时间、计时器状态（运行中/暂停）、开始时间戳

**应用重启恢复**：
1. 应用启动时检测是否存在未完成的番茄钟状态
2. 如果存在，弹出对话框："检测到未完成的番茄钟，是否继续？"
3. 用户选择：
   - "继续"：恢复计时器状态，继续倒计时
   - "放弃"：清除状态，记录为"放弃"

**系统休眠/睡眠恢复**：
1. 系统恢复时，检测休眠时长（当前时间 - 最后记录时间）
2. **自动扣除休眠时长，继续计时**（无需用户确认，因为认真工作时电脑息屏休眠是正常的）
3. 如果计算后剩余时间 ≤ 0，视为已完成，直接记录并发送通知

**用户反馈**：
- 休眠恢复后静默继续计时，不打断用户
- 如果番茄钟在休眠期间完成，显示通知告知用户

#### 3.2.1 启动番茄钟

**用户操作流程**：

1. 用户点击"开始专注"按钮
2. 界面切换到专注模式（只显示计时器和当前任务）
3. 计时器开始倒计时（默认 25 分钟）

**用户反馈**：
- 计时器显示剩余时间（MM:SS 格式）
- 播放提示音（短促的"开始"音效）
- 发送系统通知："开始专注：[任务名称]"

#### 3.2.2 暂停番茄钟

**用户操作流程**：

1. 用户点击"暂停"按钮
2. 计时器暂停
3. 界面显示"已暂停"状态

**暂停时长限制**：
- 暂停超过 **30 分钟**后，系统显示警告提示（不弹窗打扰）
- 暂停超过 **2 小时**后，系统自动将本次番茄钟标记为"放弃"
- 暂停时长会被记录，用于后续分析

**用户反馈**：
- 计时器停止倒计时
- 暂停按钮变为"继续"
- 显示已暂停时长

#### 3.2.3 继续番茄钟

**用户操作流程**：
1. 用户点击"继续"按钮
2. 计时器继续倒计时

**用户反馈**：
- 计时器恢复倒计时
- 继续按钮变为"暂停"

#### 3.2.4 放弃番茄钟

**用户操作流程**：
1. 用户点击"放弃"按钮
2. 弹出确认对话框："确定放弃本次专注吗？"
3. 对话框包含**可选**的原因选择（快捷标签 + 自定义输入）
4. 用户可选择：
   - 选择原因后点击"确定"
   - 直接点击"跳过，直接放弃"
5. 本次番茄钟记录为"中途放弃"（原因可为空）
6. 将任务与放弃原因（如有）匹配保存，供后续 AI 分析
7. 计时器重置

**快捷原因标签**：
- 临时有事
- 被打断
- 任务完成
- 不想做了
- 其他（可输入自定义原因）

**用户反馈**：
- 底部 Toast 提示"专注已中断"
- 本次专注记录被保存（状态为"放弃"，原因可为空）
- 任务状态保持不变（仍为"进行中"或"待办"）

#### 3.2.5 番茄钟完成

**用户操作流程**：
1. 番茄钟倒计时结束（自动归零）
2. 播放提示音（"完成"音效）
3. 显示"专注完成！"提示
4. 进入休息状态（短休息 5 分钟）
5. 自动记录本次专注（关联任务、时长、状态"完成"）

**用户反馈**：

- 界面切换颜色主题（专注色 → 休息色）
- 发送系统通知："专注完成！休息 5 分钟"
- 显示本次专注的简要统计

#### 3.2.6 休息时间

**用户操作流程**：
1. 进入休息状态
2. 计时器显示剩余休息时间（默认 5 分钟）
3. 用户可选择：
   - 等待休息结束（自动切换到下一个番茄）
   - 手动结束休息（跳过）
   - **暂停休息**（暂停后可继续）
   - **延长休息**（+5 分钟按钮）
4. 休息结束后，可开始下一个番茄

**休息时间规则**：
- 休息时间**可暂停**，暂停后可继续
- 跳过休息会被记录，用于后续分析
- 延长休息每次 +5 分钟，最多延长 3 次（共 20 分钟短休息）

**用户反馈**：
- 界面颜色与专注时段不同
- 播放提示音（休息结束音效）
- 发送系统通知："休息结束，开始下一个番茄"

#### 3.2.7 长休息

**用户操作流程**：
1. 系统检测到已完成 4 个番茄
2. 下一个休息自动变为长休息（15 分钟）
3. 长休息界面与短休息类似，但时间更长

**长休息计数规则**：
- 番茄计数**每天 0 点重置**
- 跨天时，前一天的番茄计数不累计到新的一天

**用户反馈**：
- 界面标注"长休息"
- 播放提示音与短休息不同

#### 3.2.8 番茄钟与任务切换

**运行中任务切换限制**：
- 番茄钟运行中**禁止切换**到其他任务
- 如需切换任务，必须先放弃当前番茄钟
- 放弃后可选择新任务开始新的番茄钟

### 3.3 习惯养成

> **科学依据**：研究表明习惯养成平均需要 **66 天**（范围 18-254 天），而非常说的 21 天。偶尔中断一天不会显著影响长期习惯形成，关键是立即恢复。

#### 3.3.1 习惯创建

**用户操作流程（简化版）**：
1. 用户点击"新建习惯"按钮
2. **快速创建**（默认模式）：
   - 输入**习惯名称**（必填）
   - 选择**习惯类型**：是/否型（默认）| 计数型 | 时长型
   - 点击"创建"即可完成
3. **高级选项**（点击"更多设置"展开）：
   - **目标值**（计数型/时长型必填）
   - **频率规则**：每天（默认）| 每周 X 次 | 特定几天
   - **图标和颜色**（可选）
   - **关联番茄钟**（可选，时长型习惯可用）
   - **提醒设置**（可选）
4. 点击"保存"
5. 习惯出现在习惯列表中

**默认值（降低用户决策成本）**：
- 习惯类型：是/否型
- 频率：每天
- 图标：根据习惯名称智能推荐（如包含"阅读"→📖，包含"运动"→🏃）
- 颜色：随机分配预设色

**习惯类型修改规则**：
- 习惯类型在**无打卡记录时可以修改**（从"是/否型"改为"计数型"等）
- 一旦有打卡记录，类型**不可更改**（避免历史数据不一致）
- 如需更改已有打卡记录的习惯类型，需删除习惯后重新创建
- 计数型目标值上限：**1000**
- 时长型目标值上限：**1440 分钟**（24小时）

**用户反馈**：
- 创建成功后习惯立即显示
- 时长型习惯可选择关联番茄钟，自动记录

#### 3.3.2 每日打卡

**用户操作流程**：
1. 用户进入"今日习惯"页面
2. 显示今日需要完成的习惯列表
3. 根据习惯类型进行打卡：
   - **是/否型**：点击习惯卡片即完成打卡
   - **计数型**：点击"+"增加计数，达到目标后自动完成
   - **时长型**：
     - 手动输入时长，或
     - 关联番茄钟时，系统自动累计当日专注时长
4. 打卡成功后显示完成动画

**用户反馈**：
- 完成打卡时播放轻微音效和动画
- 显示当前连续天数
- 计数型/时长型显示进度条（如 5/8 杯、45/60 分钟）

#### 3.3.3 跳过/请假

**用户操作流程**：
1. 用户在习惯卡片上点击"跳过"按钮（明显显示在卡片右侧）
2. 弹出确认对话框，显示剩余跳过次数
3. 用户确认跳过
4. 该习惯今日标记为"已跳过"

**跳过规则**：
- **每天习惯**：每月最多跳过 **3 次**（可在习惯设置中自定义 1-5 次）
- **每周 X 次习惯**：每周最多跳过 **1 次**
- 跳过不中断连续天数，但也不增加
- 跳过次数每月 1 日 **00:00（本地时间）** 自动重置（每天习惯），每周一 00:00 重置（每周习惯）
- 超过跳过限制时，按钮置灰，tooltip 显示"本月/本周跳过次数已用完"
- "每周 X 次"习惯：跳过的天数不计入应完成天数，周目标保持不变

**用户反馈**：
- 跳过的习惯显示为黄色/灰色状态
- 显示"已跳过"标签
- 显示本月剩余跳过次数

#### 3.3.4 习惯编辑与删除

**用户操作流程（编辑）**：
1. 用户点击习惯卡片进入详情
2. 点击"编辑"按钮
3. 修改习惯信息（名称、目标、频率等）
4. 点击"保存"

**用户操作流程（删除）**：

3. 用户确认删除
4. 习惯被归档（软删除），历史数据保留

**用户反馈**：
- 编辑成功后立即生效
- 删除后习惯从列表移除，但可在"已归档"中查看历史

#### 3.3.5 习惯统计

**连续天数（Streak）计算规则**：

| 当日状态 | 连续天数变化 |
|---------|-------------|
| 完成 | +1 |
| 跳过（请假） | 保持不变（不中断也不增加） |
| 未完成 | 归零，记录"最佳连续天数" |

**每周 X 次习惯的连续计算**：
- 按周计算，每周达标则周连续 +1
- 如"每周 3 次"：本周完成 3 次及以上，周连续 +1

**习惯强度评分（0-100%）**：
- 基于近 **30 天**的完成率计算
- 采用指数平滑算法，近期权重更高（衰减系数 α=0.95，每天衰减 5%）
- 新创建的习惯初始强度为 **0%**，第一次完成后开始计算
- 达到 **80%** 以上视为"习惯已养成"
- 达到 **66 天**连续完成，显示"🏆 习惯养成成功"徽章和进度（如"32/66天"）

**用户可查看的统计数据**：
- 当前连续天数
- 最佳连续天数
- 习惯强度（百分比）
- 本月完成次数 / 应完成次数
- 总完成次数
- 打卡热力图（类似 GitHub 贡献图）

#### 3.3.6 习惯提醒

**用户操作流程**：
1. 用户在创建/编辑习惯时设置提醒时间
2. 到达提醒时间时，系统发送通知
3. 用户点击通知可直接跳转到打卡页面

**用户反馈**：
- 通知内容："该完成「习惯名称」了"
- 点击通知直接进入打卡

#### 3.3.7 习惯与番茄钟关联

**适用场景**：时长型习惯（如"每日专注 2 小时"、"学习英语 30 分钟"）

**用户操作流程**：
1. 创建时长型习惯时，选择"关联番茄钟"
2. 可选择关联条件：
   - 关联所有番茄钟
   - 关联特定清单/标签的番茄钟
3. 当日完成的番茄钟时长自动累计到习惯进度
4. 达到目标时长后自动完成打卡

**关联计算规则**：
- 只计算状态为"**完成**"的番茄钟时长（放弃的不计入）
- 跨天番茄钟（如 23:50 开始，00:15 结束）按**开始时间**归属当天
- 同一个番茄钟**可以**同时计入多个关联习惯
- 目标超额完成时显示："60/30分钟 ✓(+30)"

**用户反馈**：
- 习惯卡片显示"已关联番茄钟"标识
- 进度自动更新，无需手动打卡

### 3.5 AI 智能分析

#### 3.5.1 任务创建时的智能建议

**触发条件**：用户开始创建任务，输入标题后

**AI 建议触发规则**：
- 防抖机制：用户停止输入 **500ms** 后触发
- 最少输入 **3 个字符**
- AI 调用超时时间：**3 秒**，超时后静默失败，不阻塞用户
- 需至少 **10 个已完成任务**的历史数据才启用建议，否则显示"完成更多任务以获得个性化建议"

**用户操作流程**：
1. 用户在任务创建对话框中输入标题
2. 系统自动分析（后台）
3. 如果有建议，显示"智能建议"按钮或区域
4. 用户点击"查看建议"
5. 系统显示建议内容：
   - 建议的任务描述
   - 建议的标签
   - 建议的预估番茄数
   - 建议理由
6. 用户点击"应用建议"
7. 表单自动填充建议内容
8. 用户可手动调整
9. 用户点击"保存"

**用户反馈**：
- 建议以折叠卡片形式显示，不占太多空间
- 用户可选择忽略建议
- 如果没有足够的历史数据，不显示建议

#### 3.5.2 每日计划建议

**触发条件**：用户点击"规划今日"或每日首次打开应用

**用户操作流程**：
1. 用户点击"规划今日"按钮
2. 系统分析：
   - 今日截止的任务
   - 用户的历史习惯
   - 优先级排序
3. 显示建议的今日任务清单
4. 用户可：
   - 一键添加所有建议任务
   - 选择性添加部分任务
   - 手动调整建议内容

**用户反馈**：
- 建议清单以可勾选列表形式显示
- 用户可查看每条建议的理由
- 添加后显示在任务列表中

#### 3.5.3 任务拖延提醒

**触发条件**：某个任务的截止时间被后延 **3 次以上**

**用户操作流程**：
1. 系统检测到任务截止时间被多次后延
2. 在任务卡片上显示警告标识
3. 用户悬停或点击标识

5. 显示拆分方案
6. 用户可选择：
   - 一键拆分
   - 忽略建议

**建议抑制规则**：
- 同类建议被用户忽略/拒绝 **3 次**后，**7 天内**不再提示该类建议

**用户反馈**：
- 警告标识显眼但不干扰
- 拆分方案合理
- 用户可选择接受或忽略

#### 3.5.4 效率下降提醒

**触发条件**：近 3 天专注时长比上周平均下降 30% 以上

**用户操作流程**：
1. 系统检测到效率下降
2. 在仪表盘显示通知
3. 用户点击通知
4. 显示分析：
   - 本周 vs 上周对比

   - 改进建议
5. 用户可关闭通知

**用户反馈**：
- 通知不强制弹窗，以卡片形式显示
- 分析内容简明扼要
- 用户可查看详细报告

#### 3.5.5 手动生成分析报告

**用户操作流程**：
1. 用户点击"生成分析报告"按钮
2. 选择时间范围（今天/本周/本月/自定义）

   - 同步路径（如 `/tracker/`）
   - 测试连接按钮
3. 用户输入 WebDAV 配置信息
4. 点击"测试连接"
5. 系统验证 WebDAV 服务器是否可访问
6. 显示测试结果（成功/失败）
7. 用户点击"保存"
8. 配置生效

**用户反馈**：

- 测试结果明确（✅ 成功 / ❌ 失败 + 原因）
- 保存后加密存储

**用户操作流程（手动同步）**：
1. 用户进入设置页面 → "云同步"
2. 显示同步状态：
   - 最后同步时间
   - 本地修改时间
   - 云端修改时间
3. 用户点击"立即同步"按钮
4. 系统执行同步：
   - 压缩本地数据库文件为 ZIP
   - 上传到 WebDAV 服务器
   - 显示同步进度（上传中/下载中）
5. 同步完成后，显示结果

**用户反馈**：
- 同步过程显示进度条
- 同步成功后显示"同步完成"通知
- 同步失败后显示具体错误原因
- 如果云端数据较新，提示用户是否覆盖本地数据

**用户操作流程（自动同步）**：
1. 用户在设置中启用"自动同步"
2. 设置同步间隔（最小 **15 分钟** / 每 30 分钟 / 每 1 小时 / 每 2 小时）
3. 系统后台定期执行同步
4. 同步不阻塞用户操作
5. 同步完成后通知用户

**同步与番茄钟**：
- 同步在**后台执行**，不中断前台操作
- 番茄钟运行中允许执行**上传**，不影响计时器
- 番茄钟运行中如检测到需要**下载/替换本地数据库**（例如云端较新且用户选择使用云端），系统将替换动作**延迟到安全点**（番茄钟结束/放弃后）再执行
- 同步替换数据库文件后，仅刷新数据库相关数据（任务/统计/习惯等）；计时器状态以本地持久化为准，不被同步覆盖

**用户反馈**：
- 自动同步不弹窗打扰用户
- 同步完成后在状态栏显示图标提示
- 点击状态栏图标可查看同步详情

**用户操作流程（冲突解决）**：
1. 系统检测到本地和云端数据有冲突
2. 显示冲突对话框：
   - 本地修改时间：2024-01-15 14:30
   - 云端修改时间：2024-01-15 16:45
3. 用户选择：
   - 使用本地版本
   - 使用云端版本
4. 系统执行用户选择

**冲突检测规则**：
- 先比较修改时间戳
- 时间相同时比较内容 hash
- 同步作为**原子操作**，任一步骤失败则整体回滚
- 大数据库同步超时时间：**5 分钟**（可配置）
- 冲突解决前自动创建本地备份，保留最近 **10 个**冲突备份

**非 HTTPS 连接**：
- 使用非 HTTPS 连接时显示**强烈警告**
- 用户确认后可继续使用（适用于局域网场景）

**用户反馈**：
- 冲突对话框清晰显示两个版本的修改时间
- 用户选择后立即同步
- 冲突解决后显示通知

**后台静默同步**：
1. 用户操作时（如创建任务、完成任务）
2. 系统在后台触发同步
3. 同步不影响用户当前操作
4. 同步完成后静默更新数据
5. 用户界面保持响应

**用户反馈**：
- 同步过程中用户可正常使用应用
- 同步完成后数据自动刷新
- 同步不中断用户操作

## 5. 非功能性需求

### 5.1 性能需求
- 应用启动时间不超过 3 秒
- 任务列表加载不超过 1 秒
- 图表渲染不超过 2 秒
- 计时器准确度误差不超过 1 秒（使用系统时间戳计算剩余时间，而非累加间隔）
- 内存占用不超过 100MB
- 空闲状态 CPU 占用不超过 1%
- 计时器运行时 CPU 占用不超过 5%
- 图表渲染时 CPU 峰值不超过 30%（持续时间 < 2 秒）

### 4.2 可用性需求

- 支持离线使用（任务管理、番茄钟、查看历史数据）
- AI 功能需联网，失败时不影响核心功能
- 首次使用提供简洁引导（见 4.5 首次使用引导）
- 离线模式时在状态栏显示离线图标

### 4.3 数据存储需求
- 所有数据本地存储
- 数据定期自动备份（每日自动备份，保留最近 7 份）
- 备份文件存储在用户数据目录的 backups 文件夹
- 支持导出数据（JSON 和 CSV 格式）
- 支持从 JSON 备份导入数据
- **支持 WebDAV 云同步**（可选）
- 同步数据压缩后上传
- 下载后自动解压同步
- 冲突解决前自动创建本地备份

### 4.4 兼容性需求
- 支持 Windows 10 21H2 及以上
- 支持 Windows 11
- 支持高 DPI 显示
- 支持暗色/亮色主题切换

### 4.5 首次使用引导

**引导流程**：
1. 首次启动应用时，显示欢迎页面
2. 引导步骤：
   - 步骤 1：创建第一个清单（可跳过，使用默认"收集箱"）
   - 步骤 2：创建第一个任务
   - 步骤 3：体验番茄钟（可跳过）
   - 步骤 4：配置 AI 功能（可跳过）
3. 引导完成后进入主界面
4. 用户可在设置中重新查看引导

### 4.6 空状态处理

| 场景 | 显示内容 |
|-----|---------|
| 无任务 | 插图 + "还没有任务，点击上方添加你的第一个任务" |
| 无清单 | 插图 + "创建一个清单来组织你的任务" |
| 无习惯 | 插图 + "创建一个习惯开始培养好习惯" |

| 搜索无结果 | "未找到匹配的任务" |

### 4.7 错误状态处理

| 错误类型 | 用户提示 |
|---------|---------|
| AI 调用失败 | "AI 服务暂时不可用，您可以继续手动操作" |
| 数据库错误 | "数据保存失败，请重试" |

| 网络不可用 | 状态栏显示离线图标，AI 功能按钮置灰 |

### 4.8 异常场景处理

#### 4.8.1 任务模块异常

| 异常场景 | 处理方式 |
|---------|---------|
| 任务标题为空或仅空白字符 | 阻止创建，输入框边框变红，无提示文字（避免干扰） |
| 任务标题超长（>500字符） | 自动截断到500字符 |
| 删除已关联番茄钟的任务 | 正常删除，番茄钟记录保留（task_id 置空） |
| 删除有子任务的任务 | 级联删除所有子任务 |
| 撤销删除超时（>10秒） | Toast 消失，删除不可撤销 |
| 批量删除大量任务（>50个） | 显示进度条，后台异步执行 |
| 拖拽排序失败 | 回滚到原位置，显示"排序失败，请重试" |

#### 4.8.2 番茄钟模块异常

| 异常场景 | 处理方式 |
|---------|---------|
| 应用崩溃/意外关闭 | 重启后显示恢复对话框，用户选择继续或放弃 |
| 系统休眠/睡眠恢复 | 自动扣除休眠时长，继续计时（无需确认，认真工作时息屏是正常的） |
| 休眠期间番茄钟完成 | 自动记录为完成，发送通知告知用户 |
| 系统时间被手动修改 | 使用持久化的开始时间戳重新计算剩余时间 |
| 计时器状态写入失败 | 重试3次，失败后显示警告但不中断计时 |
| 关联的任务被删除 | 番茄钟继续运行，完成后记录 task_id 为空 |
| 暂停超过30分钟 | 显示警告提示（不弹窗打扰） |
| 暂停超过2小时 | 自动放弃本次番茄钟，显示通知 |
| 通知权限被拒绝 | 使用应用内弹窗替代系统通知 |

#### 4.8.3 习惯模块异常

| 异常场景 | 处理方式 |
|---------|---------|
| 同一天重复打卡 | 更新已有记录而非创建新记录（UNIQUE约束） |
| 跳过次数超过月限制 | 按钮置灰，tooltip显示"本月跳过次数已用完" |
| 打卡日期超出范围（未来日期） | 阻止操作，显示"不能为未来日期打卡" |
| 补打过去日期（>14天前） | 阻止操作，显示"只能补打最近14天的记录" |
| 习惯被归档后打卡 | 阻止操作，显示"请先取消归档" |
| 关联番茄钟的习惯手动打卡 | 允许，但显示提示"已有自动打卡记录，手动打卡将覆盖" |
| 删除有大量打卡记录的习惯 | 显示确认对话框，说明将删除所有相关记录 |

#### 4.8.4 数据同步异常

| 异常场景 | 处理方式 |
|---------|---------|
| WebDAV 服务器无响应 | 超时30秒后显示"连接超时，请检查网络或服务器地址" |
| WebDAV 认证失败 | 显示"用户名或密码错误" |
| 同步过程中断网 | 回滚本次同步，显示"同步中断，请检查网络后重试" |
| 云端文件损坏/格式错误 | 显示"云端数据格式异常，请联系支持"，不覆盖本地 |
| 本地数据库损坏 | 显示"本地数据异常"，引导用户从云端恢复或重置 |
| 同步文件过大（>100MB） | 显示"数据量过大，请清理历史数据后重试" |
| 并发同步冲突 | 使用锁机制，后发起的同步等待或取消 |
| 自动同步静默失败 | 状态栏显示警告图标，不弹窗打扰用户 |

#### 4.8.5 系统级异常

| 异常场景 | 处理方式 |
|---------|---------|
| 磁盘空间不足 | 显示"存储空间不足，请清理后重试" |
| 数据库迁移失败 | 记录错误日志，回滚到上一版本，显示"升级失败，请重启应用" |
| 内存不足 | 自动释放图表缓存，必要时提示用户重启 |
| 应用启动失败 | 显示错误对话框，提供"重置数据"和"查看日志"选项 |
| 窗口最小化时计时 | 计时器正常运行，使用系统通知提醒 |
| 多实例运行 | 检测到已有实例时，激活已有窗口而非启动新实例 |
