# 跨任务学习聚合优化

## 问题描述

同样的任务结构（"去找+人名"），但相似任务的建议策略完全不同：

| 任务 | 建议结果 |
|------|----------|
| 去找宁东 | 8条学习建议（衣服、充电宝、学生证、身份证等）+ AI后台补充 |
| 去找查哥 | 仅3条AI建议（联系查哥、确认见面地点、安排见面时间），无学习建议 |

## 根本原因

学习数据（`subtask_learn_log`）按 `taskTitle + projectId` 独立存储。计算置信度时，只查当前任务的关键词匹配，没有跨相似任务聚合。

`keyword_clusters` 表可能已有跨任务语义聚类能力，但 `confidenceScorer` 没有利用它。

## 预期行为

- "去找查哥"应该能利用"去找宁东"的历史学习数据
- 跨任务语义相似性应触发学习数据的聚合回溯
- 新任务不应完全依赖AI，而应继承同类任务的学习成果

## 影响

- 新任务或历史空白任务的建议质量显著低于有过相似任务学习的任务
- 用户感觉建议不稳定、不可预测

## 修复方向

1. **置信度计算时回溯同语义簇任务**：当单任务匹配不足时，查询 `keyword_clusters` 中同簇其他任务的 `subtask_learn_log`
2. **Sibling task 学习迁移**：参考同项目下其他任务的已接受建议作为冷启动
3. **通用模式提前绑定**：像"去找XX"这类任务结构，在 `patternMatcher` 中应该有更早的匹配路径
