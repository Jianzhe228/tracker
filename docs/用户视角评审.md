# 用户视角评审 — Smart Focus Tracker v2.4.0

> 审查日期：2026-06-26  
> 审查范围：全功能（默认 Core User Review 模式）  
> 目标用户：以中文为主的知识工作者，追踪专注时间、管理日常任务

---

## Executive Read

- 核心循环（创建任务 → 启动计时 → 查看统计）体验流畅，定时器状态透传到标题栏、托盘、任务列表的设计明显高于同类工具。
- 最大摩擦点：**AI 功能发现成本极高**——新用户不知道有 AI，配置 AI 需要技术知识，配置完也不知道效果。AI 是最核心差异化功能，但对新用户几乎不可见。
- Dashboard 作为默认落地页，对零数据的新用户毫无价值（7 张图全是"暂无数据"），是第一印象损伤。
- 设置页把 9 个区域堆成一个长滚动页，找任何选项都需要全文扫描。
- 建议以最小改动优先级排列：首页默认跳 → AI 引导 → 设置分区导航 → 其余 P2。

---

## What Works

1. **定时器状态透传**：计时进行时，窗口标题变为 `25:00 · 任务名 — Smart Focus Tracker`，托盘 tooltip 跟随分钟更新，最小化时仍可在任务栏看剩余时间。这是 native-first 做得比 Electron 系竞品（Toggl Track、Forest）好的少数点之一。证据：`src/App.vue:265-302`。

2. **软删除 + 10 秒撤销**：任务删除后 10 秒内可撤回，撤销提示浮动在底部不打断操作流。证据：`src/components/AppFeedbackLayer.vue`，`deletedAt` 字段设计参见 `docs/数据库设计说明.md`。

3. **AI 建议降级链**：本地置信度 ≥ 0.7 纯本地，0.3–0.7 本地即时 + AI 后台补充，< 0.3 才调 AI。AI 未配置时建议面板显示"配置 AI 后可获得智能拆解建议"并附跳转链接，不崩溃、不报错。证据：`src/views/TasksView.vue:2952-2957`。

4. **任务视图实时统计栏**：页头四格分别显示"预计时间、待完成任务、已用时间、已完成"，不需要去 Dashboard 就能看今日进度。证据：`src/views/TasksView.vue:1850-1868`。

5. **快捷键录制 UI**：点击"录制"后按键即生效，每个快捷键可独立设为"应用内/全局"模式，逻辑清晰。证据：`src/views/SettingsView.vue:238-301`。

## Findings

### P1：Dashboard 是零数据用户的第一印象，价值为负

- **User moment**：用户首次安装，启动应用，等待它指引下一步。
- **Evidence**：Router 默认路由 `/` 对应 `DashboardView`。空数据时 7 张图全部渲染为灰色矩形"暂无数据"占位块。`src/views/DashboardView.vue:886-975`。
- **Impact**：用户看到一个全灰的统计页，不知道下一步做什么，没有任何"创建第一个任务"的引导。最大行动暗示只有侧边栏里的"今天"，且侧边栏展开才可见（折叠状态只有图标）。
- **Frequency**：影响每一个新用户的第一次开启。
- **Severity**：P1
- **Recommendation**：将首次启动（任务数为 0）时的默认路由改为 `/tasks/today`，并在任务列表空态加一行"按 Enter 添加你的第一个任务"的欢迎提示。Dashboard 对有数据的回访用户才有价值。
- **Confidence**：High

---

### P1：AI 功能对新用户完全隐形

- **User moment**：用户添加任务"准备年终述职报告"，不知道可以自动拆解子任务。
- **Evidence**：AI 建议面板只在选中任务且 `currentSuggestionPanel.requested` 为 true 时出现（`src/views/TasksView.vue:2858`）。未配置 AI 时建议面板只显示一行小字。用户首次进入设置，需要手动向下滚动过：通用 → 番茄钟 → 快捷键 → 云同步 → 通知 → 子任务模板，才能看到"AI"配置区。
- **Impact**：AI 子任务拆解是本产品相对 Todoist、Things 最强的差异化点（本地+AI 混合召回）。但没有功能说明、没有模型选择器（只有文本框填模型名）。新用户不会发现，也不知道填什么。
- **Frequency**：影响所有尚未配置 AI 的用户（含全部新用户）。
- **Severity**：P1
- **Recommendation**：
  1. 在建议面板的"前往设置"链接旁加一句"可自动拆解子任务"说明其价值。
  2. 设置页 AI 区加一个常用 provider 的快速选项（OpenAI / DeepSeek / Ollama），选中后自动填 endpoint，用户只需粘贴 API Key。
  3. AI 配置区的错误信息用结构化提示（"Key 格式错误" / "模型不存在" / "超时"）替代 `String(err)` 直接展示。
- **Confidence**：High

---

### P1：设置页 9 区垂直堆叠，无导航

- **User moment**：用户想调整番茄钟时长，或者找"子任务模板"在哪里。
- **Evidence**：`src/views/SettingsView.vue` 是 47KB 的单组件，所有设置区域（通用、番茄钟、快捷键、云同步、通知、子任务模板、AI、关于与更新、数据管理）都是顺序 `<section>` 标签，无侧边索引、无 tab 切换、无锚点跳转。
- **Impact**：重度用户调整设置时需要记忆各区位置，每次都全文扫描。设置项越多，问题越严重。
- **Frequency**：每次访问 Settings 都发生。
- **Severity**：P1
- **Recommendation**：在设置页左侧加一个锚点目录（固定定位，点击跳转到对应 section），或改为侧边 tab 导航。最轻量的实现是给每个 `<section>` 加 `id`，页面顶部加跳转链接栏。
- **Confidence**：High

---

### P2：AI / WebDAV 设置有隐式"保存"陷阱

- **User moment**：用户在 AI 设置里填完 Endpoint 和 API Key，直接关闭设置页。
- **Evidence**：AI 设置用 `v-model.trim="settingsStore.ai.endpoint"` 直接绑定到 store 字段（内存立即更新），但数据库持久化需要显式点击"保存"按钮，触发 `saveAiSettings()` → `settingsStore.updateAi()`。`src/views/SettingsView.vue:462-469, 907-926`。
- **Impact**：用户以为填了就保存了（其他主流 app 的 settings 都是即时生效），重启后发现 API Key 消失，只会以为是 bug。
- **Frequency**：任何首次配置 AI 或 WebDAV 的用户都可能踩到。
- **Severity**：P2（数据在内存存在，只是 restart 后丢失）
- **Recommendation**：改为 `watch` 在 store 字段变化时 debounce 自动持久化，不要求手动点"保存"。如果维持手动保存语义，则用未保存状态标记（输入框边框变色）提醒用户。
- **Confidence**：High

---

### P2："全部"视图的输入框行为切换不可见

- **User moment**：用户从"今天"切到"全部"，想添加一个不属于今天的任务。
- **Evidence**：`taskEntryValue` computed 的 setter：`isAllTasksView` 为 true 时改变 `searchQuery`，否则改变 `title`（`src/views/TasksView.vue:135-143`）。"全部"视图的 `placeholder` 变为"搜索任务…按 Enter 搜索"，Enter 触发搜索而非创建任务（`src/views/TasksView.vue:1884-1887`）。
- **Impact**："全部"视图没有入口创建新任务。用户如果习惯用"全部"作为主视图，必须切换到"今天"或项目视图才能新建。同一输入框在两个视图行为不同，增加心智切换成本。
- **Frequency**：高频使用"全部"视图的用户每次都会受影响。
- **Severity**：P2
- **Recommendation**："全部"视图的搜索框保持纯搜索，在列表上方单独加一个"+ 添加任务"按钮。或者在搜索结果为空时展示"按 Shift+Enter 创建此任务"的快捷提示。
- **Confidence**：High

---

### P2：WebDAV 同步为纯手动，"立即同步"实际只做上传

- **User moment**：用户在 A 机器操作后切换到 B 机器，期望数据同步好了。
- **Evidence**：`syncNow()` 函数内部只调用 `webdavUpload()`，没有 download 步骤（`src/views/SettingsView.vue:412-430`）。没有任何自动同步逻辑（无 watcher、无定时器）。
- **Impact**："立即同步"按钮名字暗示双向操作，实际只上传。多设备用户在 B 机器点了也等于什么都没做，容易因此丢失数据（覆盖了较新版本）。
- **Frequency**：使用 WebDAV 同步的用户的每次跨设备操作都受影响。
- **Severity**：P2
- **Recommendation**：
  1. 将"立即同步"改名为"上传到云端"（避免歧义），或实现真正的双向 diff 同步。
  2. 提供"应用启动时自动下载"选项，可在 `appInit` 中执行。
- **Confidence**：High

---

### P2：快捷键录制超时无进度提示

- **User moment**：用户点"录制"后犹豫，不知道什么时候窗口会失效。
- **Evidence**：`setTimeout(cleanup, 10000)` 在 10 秒后静默取消录制，按钮文字仅为"按键中…"，无倒计时。`src/views/SettingsView.vue:301`。
- **Impact**：用户以为还在录制状态，超时后按键无效，需要再次点录制。
- **Frequency**：偶发，但第一次设置快捷键的用户必然踩到。
- **Severity**：P2
- **Recommendation**：用 `setInterval` 更新录制按钮显示为"按键中 (8s)"倒计时，或在超时前 3 秒显示淡出提示。
- **Confidence**：High

---

### P3：子任务模板功能放在"设置"而非任务流程中

- **User moment**：用户在任务建议面板里看到本地建议后，想保存这组建议为模板，找不到入口。
- **Evidence**：子任务模板管理在 `SettingsView.vue` 的"子任务模板"section。建议面板（`src/views/TasksView.vue:2856-2974`）没有"另存为模板"按钮。
- **Impact**：轻度阻隔了用户从"接受建议 → 形成模板"的自然学习路径。
- **Recommendation**：在建议面板的"全部添加"旁加"存为模板"按钮。
- **Confidence**：Medium

---

### P3：App 名与 UI 语言不一致

- **User moment**：用户截图分享，标题栏显示"Smart Focus Tracker"，整个 UI 是中文。
- **Evidence**：`APP_NAME = 'Smart Focus Tracker'`（`src/utils/constants.ts`），侧边栏、窗口标题均使用该值。
- **Impact**：轻微品牌不一致感，对中文用户有割裂感。
- **Recommendation**：可改为中文名（如"专注追踪"），或在设置页明确说明这是 app 名称。
- **Confidence**：High（定性观察）

## Competitor Migration View

**来自 Todoist / Things 3 的用户期望：**

- **任务快速入口**：Todoist 允许从任何视图按 `q` 快速添加，Things 有全局快捷键直接 capture。本产品有全局快捷键但需手动配置，默认无全局入口。切换成本：低偏好 → 中等配置负担。
- **重复任务**：内置重复任务（`src-tauri/src/commands/recurring.rs`）是有的，但 UI 入口在任务详情面板深处，比 Todoist 的重复任务选择器发现性差。
- **多设备同步**：来自 Todoist 的用户习惯"打开就自动同步"，本产品的 WebDAV 手动同步流程是明确的切换障碍。

**留下来的理由：**

- 本地 SQLite，无订阅费，数据自有。
- AI 拆解 + 本地学习模式（Todoist 的 AI 仅付费版且不学习个人习惯）。
- 专注时间统计深度超过 Todoist（8 维图表 vs 基础完成率）。
